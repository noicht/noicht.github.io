Sentinel is a personal project built as a serious Java learning vehicle rather than a toy CRUD app.

The goal is to use it to practice:
- clean architecture and package boundaries
- interfaces and dependency injection
- CLI-first workflows
- logging, observability and background processing
- writing maintainable code without relying on frameworks for everything

I’m treating Sentinel as a long-term engineering sandbox where I can make deliberate design decisions, document trade-offs and improve the system over time.
